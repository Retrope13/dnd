## About The Project

[![Product Name Screen Shot][product-screenshot]](https://example.com)

I have run into a common problem when starting a dnd campaign with people who have never played before: It's too hard for them to get acustomed to all of the strange rules and the even stranger character sheet. There is so much information being thrown at the player that it can be very daunting. The goal is to create an application that feels more like a player HUD in a video game than a patient intake form.

Scope:

- Use electron-forge to create a barebones application.
- Allow users to access their stats, character info, and inventory (weapons, armor, spells, etc)
- Give users the ability to save, import and export characters.
- Create a shop for players to buy and sell items.

The above information is simply setting up a baseline scope for me to attack this project from. I really want to be able to finish this project and starting in small chunks is huge for that!

## Getting Started

If everything is set up correctly (ie. using npm to install dependencies) you should be able to run npm start to get the dev spin up.

### Prerequisites

How I built it was by using

- npm

  ```sh
  npx create-electron-app dnd --template=webpack
  npm install --save-dev @babel/core @babel/preset-react babel-loader
  npm install --save react react-dom
  Then I modified the webpack.rules.js, renderer.js and created the app.jsx file
  npm start
  ```

  if you want to run it in a dev mode you do
  `npm start`

When I have an MVP I am going to use Docker to create a container so it's very easy to port across platforms
